Graph Neural Network Model Training Workflow

The GNN model training employed a two-phase automated workflow using Optuna for hyperparameter optimization followed by final model training. In Phase 1, hyperparameter optimization was performed using the Optuna framework with MedianPruner for early stopping of unpromising trials (n_startup_trials=5, n_warmup_steps=10) to minimize validation set mean squared error. The optimization explored learning rates (log-uniform [1×10⁻⁶, 1×10⁻²]), number of graph layers (2-5), number of output layers (1-3), and samples per epoch (50-300 in steps of 50) over 30 trials with 100 epochs per trial. Each trial used the AdamW optimizer with AMSGrad, gradient clipping at 0.05, and mean squared error loss on molecular features including adjacency matrices, atomic charges, and AIM descriptors (513-dimensional input: 256 amine_aim + 256 acid_aim + 1 charges) for amine, acid, and intermediate structures, along with dynamically extracted reaction condition features. In Phase 2, the final model was trained for 100 epochs using the best hyperparameters identified in Phase 1, with hardware-agnostic device selection (CUDA/MPS/CPU) ensuring reproducibility across platforms. Each optimization run was stored in a timestamped directory containing best hyperparameters (best_params.json), all trial results (all_trials.json), the trained model checkpoint (TorchScript format), optimization history and hyperparameter importance plots, final training curves, parity plots, and CSV files with predictions for train/validation/test splits including sample names, experimental ln k values, and model predictions. Model performance was evaluated using R² and mean absolute error (MAE) metrics on all three dataset splits, with complete run metadata and performance metrics saved for reproducibility.
